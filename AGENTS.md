# AGENTS — 博客写作指引

## 核心规则

在为本仓库撰写博客文章（即 `_posts/` 或 `_drafts/` 下的 `.md` 文件）时，**必须严格遵循以下写作风格档案**：

> [rules/writing-style-profile.md](./rules/writing-style-profile.md)

## 要求

1. **前置元数据**：使用 YAML Front Matter，包含 `layout`, `title`, `date`, `author`, `comments`, `categories`, `tags`，日期格式为 `YYYY-MM-DD HH:MM:SS`。
2. **文章结构**：引言（1-3 句）→ `<!-- more -->` 阅读更多标记 → 目录 `{:toc}` → 正文。正文按「前言/背景 → 基础知识 → 核心内容 → 问题与思考 → 总结」组织。
3. **语言风格**：亲切自然、口语化表达、短句为主、设问引导、自问自答。
4. **讲解方式**：由浅入深、图文并茂（图片使用 `![描述](../images/…)`）、代码示例、表格对比。
5. **格式规范**：代码块标注语言、列表规范、关键词加粗。
6. **中英混合**：中文为主，英文术语保留原样。

## 例外

如果用户显式要求偏离上述风格（如指定特定格式、语言或结构），以用户要求为准。
